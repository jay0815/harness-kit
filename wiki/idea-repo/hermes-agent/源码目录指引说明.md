# Hermes Agent 源码目录指引说明

本文档帮助开发者快速定位代码、理解文件职责，并提供针对不同目标的阅读路径。

---

## 1. 源码树（最大 3 层）

```
hermes-agent/
├── run_agent.py                   # AIAgent 类 — 核心编排器 (~4,309 LOC)
├── model_tools.py                 # 工具编排层 (~923 LOC)
├── toolsets.py                    # 工具集定义与组合 (~876 LOC)
├── hermes_state.py                # SQLite 会话存储，含 FTS5 (~3,279 LOC)
├── agent/
│   ├── conversation_loop.py       # 核心对话循环 (~4,231 LOC)
│   ├── context_engine.py          # 可插拔上下文引擎的 ABC
│   ├── context_compressor.py      # 默认上下文压缩实现
│   ├── tool_guardrails.py         # 每轮循环检测与安全护栏
│   ├── tool_executor.py           # 并发/顺序工具调度
│   ├── memory_provider.py         # 可插拔记忆后端的 ABC
│   ├── memory_manager.py          # 内置记忆 + 外部记忆的 orchestration
│   ├── prompt_caching.py          # Anthropic cache_control 注入
│   ├── error_classifier.py        # 结构化 API 错误分类
│   ├── retry_utils.py             # 抖动退避工具
│   ├── image_utils.py
│   ├── base_utils.py
│   └── ... (80+ 模块)
├── tools/
│   ├── registry.py                # 基于 AST 的自注册工具发现
│   ├── terminal_tool.py           # 终端会话管理 (~2,405 LOC)
│   ├── delegate_tool.py           # 子代理生成 (~2,801 LOC)
│   ├── environments/              # 终端后端
│   │   ├── docker.py
│   │   ├── ssh.py
│   │   ├── modal.py
│   │   └── ...
│   └── ... (90+ 工具文件)
├── gateway/
│   ├── run.py                     # GatewayRunner + LRU 代理缓存
│   └── platforms/                 # 15+ 平台适配器
│       ├── telegram.py
│       ├── discord.py
│       ├── slack.py
│       └── ...
├── plugins/
│   ├── memory/                    # 记忆提供者插件 (honcho, mem0 等)
│   ├── context_engine/            # 上下文引擎插件
│   └── model-providers/           # 推理后端插件
├── skills/                        # 内置技能
├── optional-skills/               # 较重/小众技能
├── cron/                          # 调度器
│   ├── jobs.py
│   └── scheduler.py
├── hermes_cli/                    # CLI 子命令、安装向导
├── ui-tui/                        # Ink (React) 终端 UI
├── tui_gateway/                   # TUI 的 Python JSON-RPC 后端
├── acp_adapter/                   # ACP 服务器 (VS Code / Zed / JetBrains)
├── tests/                         # ~17K 测试，分布于 ~900 文件
└── docs/                          # 文档
```

## 2. 文件职责表

### 2.1 核心编排

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `run_agent.py` | ~4,309 LOC | AIAgent 核心编排器，协调所有子系统 | `AIAgent.__init__()`, `AIAgent.run()`, `AIAgent.run_turn()` |
| `agent/conversation_loop.py` | ~4,231 LOC | 7 阶段对话循环实现 | `ConversationLoop.run_turn()` |

### 2.2 上下文管理

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `agent/context_engine.py` | 小 | 上下文引擎抽象基类 | `ContextEngine` (ABC) |
| `agent/context_compressor.py` | 中等 | 默认阈值压缩实现 | `ContextCompressor.compress()` |
| `agent/prompt_caching.py` | 中等 | Anthropic cache_control 注入 | `PromptCaching.inject_cache_control()` |

### 2.3 工具系统

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `tools/registry.py` | 中等 | 基于 AST 的自注册工具发现 | `ToolRegistry.discover()`, `ToolRegistry.register()` |
| `agent/tool_executor.py` | 中等 | 并发/顺序工具调度 | `ToolExecutor.execute_batch()` |
| `agent/tool_guardrails.py` | 中等 | 每轮工具调用安全检测 | `ToolCallGuardrailController.check()` |
| `model_tools.py` | ~923 LOC | 工具调用编排，格式化工具定义 | `ModelTools.orchestrate()` |
| `toolsets.py` | ~876 LOC | 工具集定义、组合与加载 | `ToolsetRegistry`, `compose_toolsets()` |

### 2.4 终端/后端

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `tools/terminal_tool.py` | ~2,405 LOC | 终端会话生命周期管理 | `TerminalTool.execute()`, `SessionManager` |
| `tools/environments/*.py` | 各中等 | 终端后端实现（Docker、SSH、Modal 等） | `DockerEnvironment`, `SSHEnvironment` 等 |

### 2.5 委派

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `tools/delegate_tool.py` | ~2,801 LOC | 子代理生成与隔离 | `DelegateTool.delegate()`, `DelegateTool._create_child_agent()` |

### 2.6 会话存储

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `hermes_state.py` | ~3,279 LOC | SQLite 会话存储，WAL + FTS5 | `HermesState.create_session()`, `HermesState.save_turn()` |

### 2.7 网关

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `gateway/run.py` | 中等 | 多平台网关服务 + LRU 代理缓存 | `GatewayRunner.handle_message()`, `GatewayRunner._get_or_create_agent()` |
| `gateway/platforms/*.py` | 各中等 | 平台特定适配器（15+） | `TelegramAdapter`, `DiscordAdapter`, `SlackAdapter` 等 |

### 2.8 插件

| 目录 | 职责 |
|---|---|
| `plugins/memory/` | 记忆提供者插件（honcho、mem0、hindsight、supermemory 等） |

### 2.9 CLI

| 文件 | 规模 | 职责 | 关键类/函数 |
|---|---|---|---|
| `hermes_cli/main.py` | 中等 | CLI 入口与参数解析 | `main()` |
| `hermes_cli/setup_wizard.py` | 中等 | 交互式安装向导 | `SetupWizard.run()` |

## 3. 新开发者阅读路径

### 快速入门 — 15 分钟理解 Agent 循环

```
1. run_agent.py
   └─ 浏览 AIAgent.__init__() 参数列表，了解各子系统如何注入

2. agent/conversation_loop.py
   └─ 阅读 7 阶段循环的注释（阶段1~7），理解单轮执行流程

3. tools/registry.py
   └─ 浏览自注册机制，了解工具如何被自动发现
```

### 深入探索 — 1 小时理解上下文引擎 + 工具护栏

```
1. agent/context_engine.py
   └─ 阅读 ContextEngine ABC，理解接口契约

2. agent/context_compressor.py
   └─ 阅读默认压缩实现，理解阈值策略

3. agent/prompt_caching.py
   └─ 了解 Anthropic cache_control 的注入逻辑

4. agent/tool_guardrails.py
   └─ 阅读 ToolCallGuardrailController，理解每轮安全检测

5. agent/tool_executor.py
   └─ 了解并发/顺序调度策略

6. model_tools.py
   └─ 了解工具定义如何被格式化为模型可用的 schema
```

### 插件开发路径

```
1. 开发新工具：
   tools/registry.py → toolsets.py → 参考 tools/terminal_tool.py

2. 开发新记忆后端：
   agent/memory_provider.py → agent/memory_manager.py → plugins/memory/

3. 开发新上下文引擎：
   agent/context_engine.py → agent/context_compressor.py → plugins/context_engine/

4. 开发新平台适配器：
   gateway/run.py → gateway/platforms/telegram.py

5. 开发新推理后端：
   plugins/model-providers/ → 参考现有实现
```
